什么是Hexo?

Hexo是一款基于Node.js的静态博客框架，依赖少、易于安装使用，可以方便的生成静态网页托管在GitHub和Heroku上，是搭建博客的首选框架。这里选用的是GtiHub。

获得个人网站域名

推荐阿里云

GitHub创建个人仓库

  1. 在GitHub中创建新的仓库，
  2. 仓库名：https://用户名.github.io，用户名使用GitHub账号名称代替。

安装Git

  1. 官网下载并安装
  2. 安装成功后，将Git与GitHub账号绑定
    鼠标右键打开Git Bash，设置user.name和user.email
  3. 生成ssh密钥文件 这里直接三个回车即可，默认不需设置密码
  4. 然后找到生成的.ssh的文件夹中的id_rsa.pub密钥，将内容全部复制
  5. 打开GitHub_Settings_keys页面，新建new SSH Key，将刚刚复制的内容粘贴到Key中
  6. 在Git Bash中检测GitHub公钥是否设置成功，输入ssh git@github.com

Hexo基于Node.js，
Node.js下载地址：Download | Node.js
注意安装Node.js会包含环境变量及npm的安装
检测Node.js、npm是否安装成功，在命令行中输入

安装Hexo

Hexo就是我们的个人博客网站的框架。

  1. 新建一个文件夹，如myBlog，用来存放整个Hexo框架和你以后要发布的文章

  2. 创建好后，进入文件夹中，按住shift键，右键点击打开命令行（以下所有命令都作用在myBlog文件夹中）

  3. 使用npm命令安装Hexo，然后等待安装完成

  4. 初始化博客
    创建myBlog项目

  • hexo目录结构：
    ├── node_modules：是依赖包
    ├── public #存放被解析之后的markdown、html文件
    ├── scaffolds #当您新建文章时，根据 scaffold生成文件
    ├── source #资源文件夹
    | └── _posts #博客文章目录
    └── themes #主题
    ├── _config.yml #网站的配置信息。标题、网站名称等
    ├── db.json： #source解析所得到的
    ├── package.json # 应用程序的配置信息
  1. 生成test_my_site.md文件
    source目录下有个_post目录，这个文件夹用来存放写的md也就是你的博客内容 hexo g/generate 生成静态文件（public下的html/css/js）
    如果修改文件需要重新生成静态文件
    hexo s/server 启动服务器预览

启动服务器，网站会在 http://localhost:4000 下启动。在服务器启动期间，Hexo 会监视文件变动并自动更新，无须重启服务器。

  1. 部署
    安装部署插件

hexo clean 清除public文件夹
hexo d/deploy 将网站部署到服务器上
